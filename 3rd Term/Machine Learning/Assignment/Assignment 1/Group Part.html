5. and 6. Loads and trains the k-Nearest Neighbours classifiers for k = 1,2, ,100 and valuates each classifier using the validation data set and selects the best classifier

The goal is to classify each wine as "good" or not good based on its other available characteristics. Consequently the Y variable will be a binary variable, which will be positive if the wine is classified as "good", while the input variables will be all the other wine characteristics except for the 'quality' column.

Taking into consideration the difference among the wine characteristics' scales we will perform the k-NN analysis on the z-score normalised input data. We note that the Y binary valuable remains intact.

We "train" the k-NN classifiers for k = 1,2,..., 100.

In order to evaluate each classifier we use the validation data set and calculate the classification error for each classifier.

#KNN classifier model for K=1,..,100:
scores_list = []
for k in range(1,101):
    classifier = KNeighborsClassifier(n_neighbors=k)
    
    #Training model using training dataset:
    classifier.fit(training_data_normalised_x, training_data_normalised_y)
    
    #Predicting response for validation dataset:
    validation_data_normalised_y_predicted = classifier.predict(validation_data_normalised_x)
    scores_list.append(1-metrics.accuracy_score(validation_data_normalised_y,validation_data_normalised_y_predicted,normalize=True))

#Creating dataframe for the classification errors for validation and training set
scores_list_df = pd.DataFrame(data=scores_list).rename(columns={0: "Classification error(validation data)"})
scores_list_df['k-value'] = list(range(1,101))

Evaluate whether the classifier is well-suited for the data set

We will use Sensitivity and Specificity as measures along with the estimation misclassification rate. We will plot the confusion matrix for the classifier as well as the classification_report, which provides those metrics.

Sensitivity = (true predicted yes)/(all predicted yes) Specificity = (true predicted no)/(all actual no)

The choice of how to define "yes" it depeds on which class is considered as important. In our case the good wine is the important class and thus we will calculate the metrics based on this case.

Whether the classifier is suitable for the dataset depends on the purpose of our prediction.

For example for the classifier of the 400:200:200 sampling where k = 53, looking at the recall for Good quality Wines shows that good quality wine is predicted correctly 23% of the time. The sensitivity (precision of Good quality Wine) is 62%, meaning that when a good quality wine is predicted, the prediction is correct 62% of the time.

In comparison for the classifier with K=3 in the second sampling case, the confusion matrix shows that the recall value for good quality wines predicted correctly 35% of the time, which is higher than the first classifier. However, it has a lower sensitivity of 49% and a lower specificity, namely 90%.

Thus, the first classifier would be more suitable if it is important to know that the predicted good wines are indeed good, however it still has a low sensitivity rate. While the second model would be more suitable (compared to the first model, since generally speaking has a low performance) if our goal was to predict the most of good wines as possible.

On the other hand those models would be good choice, especially the first one, if our goal was to maximise the number of for Non-Good wines who are indeed predicted as Non-Good.
